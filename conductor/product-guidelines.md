# Product Guidelines

## Brand & Prose Style
- **Tone:** Casual & Accessible. The language should be simple, clear, and easy for anyone (especially educators) to understand. Avoid jargon where possible.

## UX Principles
- **Design System:** Material Design 3 Expressive. The application should leverage MD3's expressive capabilities for a modern, fluid, and native Android feel.

## Error Handling
- **Messaging:** Empathetic & Helpful. Errors should guide the user gently toward a solution, avoiding technical details or blame, keeping the user experience positive.
